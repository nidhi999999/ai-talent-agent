## Frontend Responsibilities

- `app.vue`
  - App shell and global navigation
- `pages/index.vue`
  - Problem and flow overview
- `pages/analyzer.vue`
  - Main recruiter workflow
  - Advanced controls:
    - match weight
    - interest weight
    - min experience
    - top-K
    - require all skills
    - must-have skills
  - Ranked output + simulated chat display
- `pages/candidates.vue`
  - Candidate pool browsing and filtering

## Backend Responsibilities

- `server/api/process.post.ts`
  - End-to-end JD parsing, matching, scoring, ranking
  - Supports two run modes:
    - `ai`
    - `free-local`
- `server/api/candidates.get.ts`
  - Candidate listing endpoint with search filters
- `server/data/candidates.ts`
  - Mock talent pool (shared dataset)

## Scoring Design

### Match Score

```text
Skill Match = matched_required_skills / total_required_skills
Experience Fit = min(candidate_experience / required_experience, 1)
Match Score = (Skill Match * 0.7) + (Experience Fit * 0.3)
```

### Interest Score

- AI mode: generated by LLM prompt
- Free-local mode: deterministic heuristic from fit ratios

### Final Score

```text
Final Score = (Match Score * matchWeight) + (Interest Score * interestWeight)
```

Weights are configurable by recruiter on `/analyzer`.

## Explainability Strategy

Each ranked candidate includes:

- matched skill count
- experience and current role context
- textual reason for simulated interest
- final weighted score

This allows judges/recruiters to understand *why* a candidate is ranked.

## Trade-offs (MVP)

- Uses mock candidate data for speed and reliability
- Simulated outreach (not real external messaging)
- No auth or ATS integrations in current version
